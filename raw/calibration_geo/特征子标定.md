# 11v_feature 标定详解

## 1. 3D点怎么计算出的

### 从2D到3D的整体思路

同一3D点被多个相机/多个时刻观测到，已知每个观测相机的位姿，通过多条射线交叉求出3D坐标。

### 详细步骤

#### 第一步：一个像素点 → 一条射线

相机拍到的每个像素点 `(u, v)`，经过去畸变后投影到归一化平面得到 `pt_nms = (x_n, y_n)`。在相机坐标系下，这个点对应一条从相机光心出发的射线：

```
射线方向 = (x_n, y_n, 1.0).normalize()
```

这条射线表示：那个3D点一定在从相机光心出发、沿着这个方向延伸的半直线上。但仅凭一个相机，不知道在射线上哪个距离处。

#### 第二步：多个观测 → 多条射线 → 交叉定位

同一3D点如果在2个或更多相机（或同一相机2个时刻）中被看到，就有2条或更多射线。这些射线理论上应该交汇于一点——就是那个3D点的位置。

#### 第三步：三角化代码（SVD方法）

`TriangulatePoint()` 函数（`group_feat_online_calib.cpp:1009-1054`）用的是经典的线性三角化（DLT）：

对每个观测，射线方向 `pts[i]` 和投影矩阵 `P_i = T_ocam_w`（世界系→相机系）构造约束方程：

```
观测约束：x_i * P_i.row(2) - z_i * P_i.row(0) = 0  (第1行)
          y_i * P_i.row(2) - z_i * P_i.row(1) = 0  (第2行)
```

把所有观测的约束堆成矩阵 `svd_A`，对它做SVD分解，最小奇异值对应的右奇异向量就是齐次坐标下的3D点：

```
svd_A * [X, Y, Z, 1]^T ≈ 0
解 = V矩阵最后一列 → pt_3d = [X, Y, Z] / 1
```

#### 第四步：质量检查

三角化后还做了两道检查：
1. **深度检查**：3D点投影回每个相机，深度 `z` 必须 > 0（点在相机前方，不是后方）
2. **角度误差检查**：投影回相机后的射线方向与原始观测方向的夹角 ≤ `bearing_angle_thres`（初始标定前5°，后续1°）

#### 第五步：选择host帧和计算逆深度

三角化得到的是世界坐标系下的 `pt_world`。在BA中不直接用xyz，而是选择协方差最小的观测帧作为host帧，计算逆深度：

```
pt_host = T_host_w.R * pt_world + T_host_w.t   // 世界→host相机系
inv_depth = 1 / pt_host.norm()                   // 逆深度
```

BA优化中只需要1个参数 `inv_depth` + host帧的观测方向 `bearing_vec` 就能恢复3D点：

```
p_host = bearing_vec / inv_depth                 // host系下3D点
```

#### 位姿链怎么拼出来的

每个关键帧的相机位姿 `T_w_ocam`（世界→ocam）由两部分拼起来（`group_feat_online_calib.cpp:1346-1354`）：

```
T_w_ocam = T_w_vcs × T_vcs_ocam
         = T_w_vcs × (T_vcs_obase × T_obase_ocam)
```

其中：
- **T_w_vcs**：从DR里程计得到，代表车辆在世界系下的位姿
- **T_vcs_obase**：从初始标定文件读入，front_wide在车辆系下的固定外参（已知）
- **T_obase_ocam**：各相机相对于front_wide的相对外参（BA要优化的量，初值从标定文件读入）

所以三角化时使用的相机位姿 = DR里程计 + 初始标定外参，之后BA再微调这些位姿和外参。

## 2. 极线搜索详解

### 什么是极线约束？

核心几何原理：如果两个相机 C1、C2 有相对位姿关系，那么 C1 图像上的一个像素点 p1，对应到 C2 图像上不是"随便哪个位置"，而是一条线——极线（epipolar line）。真正的匹配点一定在这条线上。数学表达：`p2^T * F * p1 = 0`，其中 F 是基础矩阵。

### 为什么需要极线搜索？

跨相机匹配面临两个困难：
1. 鱼眼 vs 周视：视角差异极大，纯描述子匹配容易误匹配
2. 搜索空间太大：要在整张图里搜索对应点，计算量大且容易出错

极线搜索把搜索从"整张图"缩小到"一条线附近"，大幅降低误匹配率。

### 代码实现逐行解析

`MatchByEpipolarSearch()`（`keypoint_matcher.h:304-463`）：

#### Step 1：去畸变 + 投影到归一化平面

```cpp
UndistortPoints(pts_src, src_intrinsic, &pts_src_undist);   // 去畸变
PixelToNormalizePlane(src_intrinsic, &pts_src_nms);          // 像素→归一化平面
// 对目标图同样处理
```

归一化平面坐标 `(x_n, y_n)` 消除了内参K的影响，让几何计算更简洁。

#### Step 2：计算本质矩阵 E 和基础矩阵 F

已知两相机间的相对变换 `T_tar_src`（从初始外参计算）：

```cpp
R = T_tar_src.block<3,3>(0,0);   // 旋转
t = T_tar_src.block<3,1>(0,3);   // 平移

// 反对称矩阵 [t×]
t_x <<  0, -t(2),  t(1),
       t(2),    0, -t(0),
      -t(1),  t(0),    0;

E = t_x * R;                     // 本质矩阵
F = K_tar⁻¹ᵀ * E * K_src⁻¹;     // 基础矩阵
```

E 描述归一化平面上的极线约束，F 描述像素平面上的极线约束。

#### Step 3：对源图每个特征点，生成目标图上的极线

这是最关键的一步。源图特征点 `pt_nms` 在归一化平面上，对应3D空间中的一条射线。这条射线上不同深度的点投影到目标图上会落在不同位置，但这些投影点连起来就是一条线——极线。

```cpp
// 假设3D点在不同深度处
pt_max = R * (pt_nms * max_range) + t;   // 深度=100m处投影到目标相机系
pt_min = R * (pt_nms * min_range) + t;   // 深度=1m处投影到目标相机系

// 投影到目标图像素平面
pix_start = K_tar * (pt_min / pt_min.z);  // 极线起点（近处投影）
pix_end   = K_tar * (pt_max / pt_max.z);  // 极线终点（远处投影）
```

直觉理解：3D点在1m处时，目标相机看到的投影位置和100m处时不同，但它们连起来就是极线。深度未知，所以匹配点一定在这条线上的某个位置。

#### Step 4：找极线附近的候选匹配点

```cpp
for (int j = 0; j < pts_tar_undist.size(); ++j) {
    float p2l_dist = src_line.GetPointToLineDist(pts_tar_undist[j]);
    if (p2l_dist > epc_p2l_dist_threshold)  // 10像素
        continue;
    candidate_tar_pts_id.emplace_back(j);
}
```

遍历目标图所有特征点，计算它们到极线的距离。距离 ≤ 10像素的作为候选。

#### Step 5：描述子距离选最优

```cpp
for (int tar_id : candidate_tar_pts_id) {
    int dist = DescriptorDistance(tar_desc, src_desc);  // 汉明距离
    if (dist < best_match_dist) {
        best_match_dist = dist;
        best_match_id = tar_id;
    }
}
// 还要满足 dist < min_orb_dist (50)
```

在候选点中选描述子距离最小的，且必须 < 50。

#### Step 6：RANSAC剔外点（可选）

对最终匹配结果用 `cv::findFundamentalMat` 做RANSAC，剔除不符合极线约束的误匹配。

### 极线搜索 vs 纯描述子匹配对比

| | 纯KNN匹配 | 极线搜索 |
|--|--|--|
| 搜索范围 | 整张图 | 极线附近10像素 |
| 几何约束 | 无（或事后RANSAC） | 先验外参提供强约束 |
| 误匹配率 | 高（尤其跨相机） | 低 |
| 适用场景 | 同相机时序匹配 | 跨相机匹配 |

## 3. BA中 T_base_ocam 和车辆位姿怎么理解

### 三个坐标系层次

```
W (世界坐标系)   —— 真实的物理世界，XYZ固定不动
    ↑
    |  T_w_b  (车辆位姿，随车移动而变化)
    |
B (base坐标系)   —— FRONT_WIDE相机的ocam坐标系，固定在车上
    ↑
    |  T_b_c  (相对外参，相机间的安装关系，不变)
    |
C (目标相机坐标系) —— 比如SURR_REAR的ocam坐标系
```

### 车辆位姿 T_w_b 是什么

T_w_b 就是 FRONT_WIDE相机在世界坐标系下的位姿。

代码中叫 `optim_base_poses`，参数化为 SE3（xyz平移 + 四元数旋转，共7维）。

时刻 t=0 时：车在世界坐标 (10, 5, 0) 处，朝向北。所以 T_w_b_0 把 B 系的原点映射到 W 系的 (10, 5, 0)。
时刻 t=1 时：车移动到 (10.1, 5.2, 0)，略微转弯。所以 T_w_b_1 不同。

关键：front_wide固定在车上不动，所以 T_w_b 的变化完全反映车的运动。这就是"车辆位姿"——不是抽象的"车体坐标系位姿"，而是front_wide相机在世界系下的位姿。

**初值来源**：DR里程计提供：
```cpp
frame_pkg->T_w_vcs.block<3,3>(0,0) = Intr_XYZ2R(curr_odo_.rpy_);  // DR旋转
frame_pkg->T_w_vcs.block<3,1>(0,3) = curr_odo_.t_;                // DR平移
T_w_ocam = T_w_vcs * T_vcs_ocam;
```

其中 T_w_vcs 是DR提供的车辆在世界系下的位姿，T_w_ocam 是完整的front_wide在世界系下的位姿。

BA中 T_w_b 的初值从 T_w_ocam 转换而来，优化时会被微调（修正DR的误差）。

**第一、二帧的平移被fix**，保证尺度一致性（纯视觉无法确定绝对尺度，需要DR提供）。
**相邻帧间添加 RelativePoseFactor**（DR里程计约束），防止轨迹漂移。

### 相对外参 T_b_c 是什么

T_b_c 就是目标相机坐标系相对于 FRONT_WIDE 相机坐标系的变换。

代码中叫 `optim_relative_poses`，也是 SE3（7维），但平移被fix，只优化旋转。

具体含义：把 SURR_REAR ocam 系下的一个点，通过 T_b_c 变换到 FRONT_WIDE ocam 系下。

**为什么 fix 平移？**
- 平移是两相机间的物理安装距离，出厂后不会变
- 标定关心的是旋转角度误差（roll/pitch/yaw），这是安装角度偏差
- 优化旋转（3维）比优化整个SE3（6维）更稳定，约束更强

**初值来源**：从标定文件读入：
```
T_obase_ocam = T_vcs_obase⁻¹ × T_vcs_ocam   // front_wide系 → surr_rear系
```

**FRONT_WIDE自身的相对外参为单位阵，且被 SetParameterBlockConstant（不优化）。**
**已收敛的相机外参也被 fix，不再优化。**

### BA中的完整位姿链：重投影误差计算

从 reproj_factor.h:364-378 的 ComputeResidual()，用具体数字推演：

假设时刻 t=0，FRONT_WIDE（host）和 SURR_REAR（target）同时看到一个3D点 P。

```
步骤1：组合 host 相机的完整世界系位姿
  R_w_h = R_w_b_h × R_b_c_h     (世界→front_wide_base × base→front_wide)
  t_w_h = R_w_b_h × t_b_c_h + t_w_b_h
  
  对于front_wide，T_b_c_h是单位阵（base相机到自己是单位变换）
  所以 R_w_h = R_w_b_h, t_w_h = t_w_b_h

步骤2：组合 target 相机的完整世界系位姿
  R_w_t = R_w_b_t × R_b_c_t     (世界→front_wide_base × front_wide→surr_rear)
  t_w_t = R_w_b_t × t_b_c_t + t_w_b_t
  
  对于surr_rear，T_b_c_t就是我们要优化的相对外参！

步骤3：从 host 观测恢复3D点
  p_h = bearing_vec_host / inv_depth     // front_wide系下的3D点
  p_w = R_w_h × p_h + t_w_h              // → 世界系

步骤4：将3D点投影到 target 相机系
  p_t = R_w_t^T × (p_w - t_w_t)         // 世界系 → surr_rear系

步骤5：计算残差
  residual = sqrt_info × tangent_base × (p_t.normalized() - bearing_vec_target)
  // target观测方向 vs 计算出的3D点投影方向，在单位球面上的误差
```

### 三种残差类型的物理含义

#### 类型1：TwoBaseTwoExtrinsic

- 时刻t=0 FRONT_WIDE看到点P（host），时刻t=1 SURR_REAR也看到点P（target）
- host的base位姿 ≠ target的base位姿（不同时刻）
- host的外参 ≠ target的外参（不同相机）
- 优化变量：T_b_c_h(单位阵,fix), T_w_b_0, T_b_c_surr, T_w_b_1, inv_depth

#### 类型2：OneBaseTwoExtrinsic（最重要的！）

- 时刻t=0，FRONT_WIDE看到点P（host），同一时刻 SURR_REAR也看到点P（target）
- host的base位姿 = target的base位姿（同一时刻，同一个base！）
- host的外参 ≠ target的外参（不同相机）
- 优化变量：T_b_c_front_wide(fix), T_w_b_0, T_b_c_surr(要优化!), inv_depth

这就是优化相对外参 T_b_c 的核心约束：同一时刻，同一3D点被front_wide和surr_rear同时看到，已知front_wide的位姿（同一base），通过重投影误差来修正surr_rear相对于front_wide的角度。

#### 类型3：TwoBaseOneExtrinsic

- 时刻t=0 FRONT_WIDE看到点P（host），时刻t=1 FRONT_WIDE又看到点P（target）
- 不同时刻但同一相机
- host的base位姿 ≠ target的base位姿（不同时刻）
- host的外参 = target的外参（同一相机，外参不变）
- 优化变量：T_b_c(fix), T_w_b_0, T_w_b_1, inv_depth

这就是优化车辆位姿 T_w_b 的核心约束：同一相机在不同时刻看到同一3D点，通过重投影误差来修正各时刻的base位姿。

### 优化变量总结

```
优化变量        初值来源        fix/优化        物理含义
─────────────────────────────────────────────────────────────
T_w_b (×N帧)   DR里程计        优化旋转+平移    front_wide在世界系下的轨迹
                                 (fix前2帧平移保尺度)
                                 
T_b_c (×8相机)  标定文件初值     只优化旋转        各相机相对于front_wide的角度(RPY)
                 (平移fix)       (收敛后也fix旋转)

inv_depth (×M点) 三角化计算      优化             3D点的逆深度
```

### BA的整体逻辑

- 跨相机共视 → 优化相对外参RPY（这是标定目的）
- 单相机时序 → 优化base轨迹（修正DR误差，保证几何一致性）
- DR里程计 → 提供base轨迹先验（RelativePoseFactor，防止漂移）
- 三者联合优化，互相制约，最终得到准确的相机间RPY
