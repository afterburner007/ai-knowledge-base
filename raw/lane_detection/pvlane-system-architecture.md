# PVLane 检测系统架构与原理

## 一、系统概述

PVLane 是一个基于深度学习的车道线检测系统，支持多种相机视角（单目、侧视、鱼眼）的车道线语义分割。系统采用 ONNX 推理引擎，支持 CUDA 加速，具备高效的数据处理和可视化能力。

### 3.2 单目/侧视模型架构（ResNet-Backbone + Lane Head）

**网络结构概览：**

```
输入：[B, 3, 544, 960]
         │
         ▼
┌─────────────────────────────────────┐
│        Image Backbone (ResNet)      │
│  • conv1: 7x7 卷积，stride=2        │
│  • layer1: Bottleneck x3  [64->256] │
│  • layer2: Bottleneck x4  [256->512]│
│  • layer3: Bottleneck x6  [512->1024]│
│  • layer4: Bottleneck x3  [1024->2048]│
└─────────────────────────────────────┘
         │
         ▼  多尺度特征 {P2, P3, P4}
┌─────────────────────────────────────┐
│           Mono/Side Neck            │
│  • channel_transform: 1x1 卷积降维   │
│    - 256 -> 64   (P2)               │
│    - 512 -> 96   (P3)               │
│    - 1024 -> 160 (P4)               │
│  • 特征融合 + 上采样                 │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│          PV Lane Head               │
│  • seg_convs: 分割卷积层            │
│  • stride8: 8 倍上采样到原图 1/8      │
│  • output_convs: 1x1 卷积输出        │
└─────────────────────────────────────┘
         │
         ▼
输出：[B, 272, 480]  (输入 1/2 分辨率)
```

**节点统计：**

| 类型 | 数量 |
|------|------|
| 总节点数 | 168 |
| Conv | 73 |
| Relu | 61 |
| Add | 20 |
| ConvTranspose | 2 (上采样) |
| MaxPool | 1 |
| ArgMax | 1 |

**权重通道分析：**

```
pv_mono_neck/channel_transform:
  • [64, 256, 1, 1]    <- P2 特征降维
  • [96, 512, 1, 1]    <- P3 特征降维
  • [160, 1024, 1, 1]  <- P4 特征降维

pv_lane_head:
  • seg_convs.stride8.0.0.0: [64, 1, 3, 3]  <- 深度可分离卷积
  • seg_convs.stride8.1: [32, 32, 2, 2]     <- 反卷积上采样
  • output_convs: [6, 32, 1, 1]             <- 6 类输出
```

### 3.5 模型选型原因分析

**为什么单目/侧视使用 ResNet，而鱼眼使用 VarGNet？**

#### 3.5.1 成像特性差异

| 相机类型 | 成像特点 | 对模型的影响 |
|----------|----------|--------------|
| 单目/侧视 | 透视投影，畸变小 | 标准卷积即可有效提取特征 |
| 鱼眼相机 | 球面投影，径向畸变大 | 需要大感受野适应非线性畸变 |

**鱼眼相机的成像特点：**
- **径向畸变**: 图像边缘的直线呈现弯曲
- **视野范围广**: 通常 >120°，甚至达到 180°
- **分辨率分布不均**: 中心区域分辨率高，边缘区域压缩严重

#### 3.5.2 ResNet vs VarGNet 架构对比

**ResNet (单目/侧视):**
```
┌─────────────────────────────────────┐
│  标准卷积 (3x3, 固定感受野)          │
│  • 适用于规则网格特征               │
│  • 参数效率高                       │
│  • 特征提取稳定                     │
└─────────────────────────────────────┘
```

**VarGNet (鱼眼):**
```
┌─────────────────────────────────────┐
│  可变分组卷积 (不同区域不同核)        │
│  • 中心区域：小卷积核 (高分辨率)     │
│  • 边缘区域：大卷积核 (适应畸变)     │
│  • 更大感受野：捕捉弯曲车道线        │
└─────────────────────────────────────┘
```

#### 3.5.3 设计决策原因

**单目/侧视选择 ResNet 的原因：**
1. **成熟稳定**: ResNet 是经典的 Backbone，训练收敛快
2. **参数效率**: 较少的参数量 (73 个 Conv) 适合实时推理
3. **特征复用**: 预训练权重可直接迁移
4. **输出下采样**: 2 倍下采样 (272x480) 足够满足远距离车道线检测

**鱼眼选择 VarGNet 的原因：**
1. **畸变适应**: 可变分组卷积能够适应鱼眼的非线性畸变
2. **大感受野**: 323 个节点，更多层次捕捉长距离依赖
3. **原分辨率输出**: 640x960 输出保持近距离车道的细节
4. **边缘特征增强**: 针对鱼眼边缘压缩区域有特殊处理

#### 3.5.4 任务需求差异

| 应用场景 | 单目/侧视 | 鱼眼 |
|----------|-----------|------|
| 检测距离 | 远距离 (50m+) | 近距离 (10m 内) |
| 精度要求 | 车道线类型识别 | 近距离精确轮廓 |
| 视野重点 | 前方道路 | 车辆四周盲区 |
| 输出策略 | 语义分割 (6 类/4 类) | 精细分割 |

#### 3.5.5 性能权衡

```
单目/侧视：
  优势：推理速度快、模型小、易部署
  劣势：对极端畸变适应性差

鱼眼：
  优势：畸变适应性强、近距离精度高
  劣势：模型大 (100MB)、计算量大 (323 节点)
```

> **总结**: 模型选型是根据相机成像特性 + 应用场景需求的综合权衡结果。

## 五、深度学习架构原理

### 5.1 Backbone 设计 - ResNet 特征提取器

PVLane 的单目和侧视模型采用 ResNet-50 风格的 Backbone 进行特征提取：

```
输入图像 [3, 544, 960]
    │
    ▼
┌────────────────────────────────────────────┐
│ conv1: 7x7 卷积，stride=2, padding=3        │
│ + BatchNorm + ReLU                          │
│ 输出：[64, 272, 480]                        │
└────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────┐
│ maxpool: 3x3, stride=2                      │
│ 输出：[64, 136, 120]                        │
└────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────┐
│ layer1: Bottleneck x3                       │
│ 通道：64 -> 256 (实际 3x3 卷积通道为 64)        │
│ 输出：[256, 136, 120]                       │
└────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────┐
│ layer2: Bottleneck x4                       │
│ 通道：256 -> 512 (stride=2 下采样)            │
│ 输出：[512, 68, 60]                         │
└────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────┐
│ layer3: Bottleneck x6                       │
│ 通道：512 -> 1024 (stride=2 下采样)           │
│ 输出：[1024, 34, 30]                        │
└────────────────────────────────────────────┘
    │
    ▼
┌────────────────────────────────────────────┐
│ layer4: Bottleneck x3                       │
│ 通道：1024 -> 2048 (stride=2 下采样)          │
│ 输出：[2048, 17, 15]                        │
└────────────────────────────────────────────┘
```

### 5.2 Bottleneck 结构

```
输入 F
  │
  ├─────────────────────────────────────┐
  │                                     │
  ▼                                     │
┌───────────┐                           │
│ 1x1 Conv  │  降维 (256->64)            │
│ + BN +ReLU│                           │
└───────────┘                           │
  │                                     │
  ▼                                     │
┌───────────┐                           │
│ 3x3 Conv  │  特征提取 (64->64)         │
│ + BN +ReLU│                           │
└───────────┘                           │
  │                                     │
  ▼                                     │
┌───────────┐                           │
│ 1x1 Conv  │  升维 (64->256)            │
│ + BN      │                           │
└───────────┘                           │
  │                                     │
  ▼                                     │
⊕ ◄─────────────────────────────────────┘
│ Residual Connection
│ (若维度不匹配，通过 1x1 卷积调整)
│
▼
ReLU
```

### 5.3 Neck 设计 - 多尺度特征融合

**pv_mono_neck 结构：**

```
┌─────────────────────────────────────────┐
│  channel_transform (1x1 卷积)             │
│  • P2 (256 通道) -> 64 通道               │
│  • P3 (512 通道) -> 96 通道               │
│  • P4 (1024 通道) -> 160 通道             │
└─────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────┐
│  特征金字塔融合 (FPN 风格)                │
│  • 高层特征上采样 + 逐元素相加           │

└─────────────────────────────────────────┘
```

### 5.4 Head 设计 - 车道线分割头

```
融合特征
    │
    ▼
┌─────────────────────────────────────────┐
│  seg_convs (分割卷积)                    │
│  • 3x3 卷积 + ReLU                        │
│  • 逐层提取车道线特征                    │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  stride8 (8 倍上采样)                     │
│  • 反卷积 (2x2, stride=2) x2             │
```

### 5.5 鱼眼模型 - VarGNet 架构

**VarGNet (Variable Group Network) 特点：**
- **可变分组卷积**: 不同区域使用不同的卷积核组
- **大感受野**: 适应鱼眼镜头的畸变特性
- **多尺度融合**: 13 次 Resize 操作融合多尺度特征

### 6.2 ONNX 推理引擎

```python
class ONNXInferenceEngine:
    """单例模式确保模型只加载一次"""

    _session = None  # ONNX Session 单例
    _model_path = None

    @classmethod
    def get_session(cls, onnx_path):
        if cls._session is None or cls._model_path != onnx_path:
            sess_options = ort.SessionOptions()
            sess_options.graph_optimization_level = ort.GraphOptimizationLevel.ORT_ENABLE_ALL
            cls._session = ort.InferenceSession(
                onnx_path,
                sess_options=sess_options,
                providers=['CUDAExecutionProvider', 'CPUExecutionProvider']
            )
        return cls._session
```

**关键特性:**
- **单例模式**: 避免重复加载模型
- **CUDA 加速**: 优先使用 GPU 推理
- **图优化**: `ORT_ENABLE_ALL` 级别优化

## 十、关键技术点

### 10.1 性能优化策略

| 优化项 | 实现方式 |
|--------|----------|
| 模型加载 | 单例模式，避免重复初始化 |
| 推理加速 | CUDA Execution Provider + 图优化 |
| 数据加载 | PyTorch DataLoader (多进程 num_workers=4) |
| 后处理 | 异步队列 + 后台线程 |
| 内存控制 | Queue 限制 maxsize |
| 向量化染色 | Numpy 索引操作代替循环 |

### 10.2 尺寸对齐处理

```python
# 关键修复：确保 mask 尺寸与原图对齐
if colored_mask.shape[0] != h or colored_mask.shape[1] != w:
    colored_mask = cv2.resize(colored_mask, (w, h), interpolation=cv2.INTER_NEAREST)
```

### 10.3 支持的图像格式

**输入：** `.png`, `.jpg`, `.jpeg`

**输出：**
- Mask: `.png` (单通道类别图)
- 可视化：`.jpg` (BGR 三通道)
- 原始数据：`.npy` (Numpy 格式)

## 十一、面试总结要点

### 11.1 系统设计亮点

1. **生产者 - 消费者模式**: 推理与后处理解耦，提升吞吐量
2. **单例模式**: 模型资源高效复用
3. **异步流水线**: 最大化 GPU 利用率
4. **向量化处理**: 避免 Python 循环，使用 Numpy 索引加速

### 11.2 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| Mask 与原图尺寸不匹配 | cv2.resize + INTER_NEAREST |
| 内存溢出 | Queue 限制 maxsize |
| 模型重复加载 | 单例模式 |
| 后处理阻塞推理 | 异步后台线程 |

### 11.3 可扩展性

- **新模型支持**: 扩展 `size_map` 和标签字典
- **新相机类型**: 调整 `PipelineDataset` 的几何变换逻辑
- **新输出格式**: 修改 `post_process_worker`
